# 应用安全检测（60 项）真实实测 + DOCX 报告生成

面向 Web/内网应用的「60 项应用安全检测」标准化流程：真实登录取证 + 黑盒实测 → 套用 V2 模板生成正式 DOCX/PDF 报告 → 按约定交付。适用于同一平台多环境（UAT/SIT/DEV/生产）或结构相似的新系统。

## 引用标准现行版本（每次开工先核对，务必用最新版）
下列为截至 2026-07 核实的现行版本；引用「测试依据」一律用最新版，且每次执行前重新核对是否有更新：
- **OWASP Web 安全测试指南（WSTG）v4.2** —— 旧称「OWASP 安全测试指南 V4.0」已过时，现行稳定版为 WSTG v4.2。
- **OWASP Top 10:2025** —— owasp.org/Top10 已指向 2025 版；引用请写明年份版本，勿写无版本的「OWASP TOP10」。
- **《中华人民共和国网络安全法（2025 修正，自 2026-01-01 施行）》** —— 2025-10-28 全国人大常委会通过修改决定，2026-01-01 起施行；引用应指向现行修正版。
- **GB/T 22239-2019《信息安全技术 网络安全等级保护基本要求》（等保 2.0）** —— 2025-05-30 复审结论「继续有效」，现行有效，无需更换。
- **《中华人民共和国数据安全法》（2021-09-01 施行）**、**《中华人民共和国个人信息保护法》（2021-11-01 施行）** —— 均现行有效。

## 前置输入（开工前先确认齐全）
- 目标：完整 URL（含 scheme/host/port/path），如 `https://mbmp-uat.dfmc.com.cn` 或 `http://10.124.9.253:443/fp/`
- 凭据：管理员账号 + 尽量再要一个普通用户账号（用于越权/数据隔离 PER-002/003 验证；缺则相关项标「未测试」）
- 交付去向：GitHub 仓库（如 `change1238/32` main）和/或本地目录（如 `D:\safetest`）
- 基准模板：既有 V2 DOCX 报告 + 对应改写脚本（`make_*.py`、`dump.py`）所在目录（默认 `/home/ubuntu/safetest`）
- 内网目标：确认本地是否可达、是否需 cpolar 隧道穿透

缺任何一项先用 message_user 一次性问清，不要臆测或用错凭据兜底。

## 内网穿透（仅当目标在内网/本地时）
- **优先 TCP 隧道，别用 HTTP 隧道**：cpolar HTTP 隧道会改写 Host 头、注入 `X-Forwarded-*`，很多后端网关据此拒绝 → 统一 401/502。TCP 隧道原样透传字节，请求与浏览器一致。
  ```
  cpolar tcp <IP>:<port> -config=<cpolar.yml>
  # 得到 tcp://x.tcp.cpolar.cn:PORT -> <IP>:<port>，用 http://x.tcp.cpolar.cn:PORT 访问
  ```
- 若拿到 401 且用户确认本地能正常访问 → 几乎必是 HTTP 隧道代理头问题，直接切 TCP 隧道。
- cpolar 免费版同一时刻只有一条隧道：文件服务隧道与应用隧道不能并存，按需切换。
- 交付内网结果默认「只改本地、不推 GitHub」（除非用户明确要推）。

## 阶段一：黑盒探测（curl，不需登录）
逐项取真实事实，别照抄模板旧结论：
- 端口/TLS：`http` vs `https`、端口、证书、TLS 版本（PORT）
- 响应头：HSTS / X-Frame-Options / CSP / X-Content-Type-Options / Server 版本泄露（BASIC/TRAN/VULN）
- CORS：`OPTIONS` 带 `Origin: https://evil.example` 看是否反射任意源 + `Allow-Credentials:true`（SESS-002）
- 版本/信息泄露端点：`/actuator*`、`/swagger-ui`、`/v2|v3/api-docs`、`/druid`、`version.json`（注意区分真实响应 vs SPA fallback 200）
- 前端 JS bundle：抓 `index.*.js`，grep 硬编码密钥/RSA 公私钥/AES key/内网 IP/baseURL（FRON-002/KEY-001）
- 登录接口：验证码是否开启（`captchaEnabled`）、空体提交、错误提示是否泛化（AUTH-001/005/ERR-001）
- HTTP 方法：PUT/DELETE/TRACE 是否放行（VULN）

## 阶段二：真实登录 + 认证态深测（浏览器 computer 工具）
- 用管理员凭据登录；特殊字符密码（如 `V{Fc~39m`）易被 type 动作丢字符 → 校验密码框长度，必要时用 JS 直接 set value + 触发 input 事件。
- 从 localStorage/cookie 取 Token，配合 curl 做 API 级枚举（`getInfo` 角色权限、`getRouters` 真实菜单路由、`user/list`、`role/list`、`monitor/*`）。
- 逐一取证并截图：
  - 管理控制台（应用/用户/角色管理）
  - 越权：普通用户访问管理员 URL（无普通账号则标未测试）
  - 数据脱敏：用户手机号/邮箱、业务数据税号/身份证/银行账号/地址/VIN 是否明文（DATA-002/003）；横向滚表格露出敏感列取证
  - 审计日志 schema：账号/客户端 IP/模块/操作/方法/状态/时间 + 记录数（AUD-001/002）
  - 会话：Token 存储位置、HttpOnly/Secure、登出失效、超时、每次登录换签（SESS）
- 截图存 `fp_imgs/` 等目录；黑盒结论可用 PIL 把真实 curl 输出渲染成终端风格证据图。

## 阶段三：套模板生成报告（改写脚本，别手改每格）
以既有环境 DOCX 为结构基准，写 `make_<env>.py`：
1. 全局文本替换：系统名、环境标识、域名、IP、用户/租户、技术栈、日期、凭据脚注。**慎用裸 "DEV" 之类替换**（会误伤版本号），改用精确标识串。
2. 逐项覆盖 `测试结果 / 截图说明 / 修复建议`；**`风险等级` 是项固有值一般不改**，只改测试结果。缺凭据/无法验证的项据实标「未测试」，不适用标「不涉及」。
3. 图片：按标题（caption）段落定位 blip 引用替换真实截图；先 PIL 压缩（PNG→JPEG，宽≤1300px，quality~78），避免 DOCX 膨胀（20MB→~1-2MB）。
4. 更新风险汇总表、优先级表、概述统计段、变更记录，保证「符合/不符合/未测试/不涉及」计数与逐项一致。
5. 用 `dump.py`（caps/tables 模式）核对表格索引与项→表映射；跑完 grep 残留旧标识（EUSD/mbmp/旧域名/AES-GCM/租户 等）。
6. `libreoffice --headless --convert-to pdf` 出预览，`pdftoppm` 抽几页人工核对封面、DATA/KEY 证据页、脚注是否渲染正确。

## 阶段四：交付
- GitHub：`cp` 报告进本地 clone（如 `/home/ubuntu/repo32`）→ `git add 指定文件`（**绝不 `git add .`，绝不提交 token/凭据**）→ commit 写清系统/IP/关键结论 → `git push origin main`。
- 本地：经文件服务隧道上传到用户目录，或让用户本地 `git pull`。
- 交付后：吊销会话中明文出现过的 PAT/凭据（安全提醒），standby 后续需求。

PORT 端口 / BASIC 基线 / FRON 前端 / TRAN 传输 / AUTH 身份鉴别 / PER 权限 / SESS 会话 / KEY 密钥 / ERR 异常处理 / BUS 业务 / AUD 日志审计 / DATA 数据 / VULN 常见漏洞（SQLi/XSS/弱口令等）。风险分级：严重 / 高危 / 中危。
# 附录：应用安全检测项主清单（59 项 / 13 类）

> 环境无关的检测项定义（编号/风险名称/风险等级/功能要求/风险影响/测试依据）。实测时逐项填 测试结果/截图说明/修复建议。


## PORT 端口安全（1 项）

### PORT-001 应用端口服务安全测试 ｜风险等级：严重
- 功能要求：排查应用高危端口可直接访问等风险，如排查应用或服务管理端（包含不限于网络安全设备、云平台、虚拟化平台、大数据平台、邮件管理端、通用类应用管理端等）端口是否限制直接访问、tomcat ajp协议是否做隔离等风险。 对于三级系统以及重要业务系统，管理端端口应进行IP/IP段白名单访问限制。
- 风险影响：攻击者通过高危端口对应用服务开展漏洞利用攻击行为，获取应用服务权限并造成敏感信息泄露
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》


## BASIC 基线安全（1 项）

### BASIC-001 应用软件及相关组件版本基线安全测试 ｜风险等级：高危
- 功能要求：应用软件开发框架、中间件和其他应用组件等，应满足安全基线要求
- 风险影响：低版本组件存在安全漏洞，攻击者利用漏洞可获取敏感信息或服务器应用权限
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》


## FRON 前端安全（2 项）

### FRON-001 前端JS库软件版本安全测试 ｜风险等级：中危
- 功能要求：前端应使用满足基线版本的JavaScript库版本
- 风险影响：低版本的JavaScript库存在已知安全漏洞，这些漏洞可能允许攻击者在用户浏览网页时执行恶意操作，如窃取用户数据、操纵页面内容、发起跨站脚本攻击(XSS)等
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》

### FRON-002 前端泄露风险测试（如JS） ｜风险等级：严重
- 功能要求：前端JS文件应不包含敏感信息，如内网接口、密钥、个人敏感信息、应用敏感信息等
- 风险影响：前端JS文件中的存在敏感信息可能被攻击者探测获取，造成敏感信息泄露。
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》


## TRAN 传输安全（2 项）

### TRAN-001 软件版本信息泄漏安全测试 ｜风险等级：中危
- 功能要求：应用服务不应泄露软件版本信息。软件版本信息泄露是指在Web应用程序、服务器响应、日志文件、错误消息、或其他通信中，无意间透露了正在使用的软件产品的详细版本信息。
- 风险影响：如果泄露的版本是一个已知存在严重漏洞的版本，攻击者可以迅速识别出目标系统可能存在的安全隐患，从而更容易发动攻击，获取应用服务权限，产生敏感信息泄露的风险。
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》

### TRAN-002 防重放攻击、防篡改功能安全测试 ｜风险等级：高危
- 功能要求：1、请求应具备防重放攻击防护,防止攻击者截取请求，利用重放方式持续进行漏洞探测 2、请求应具备防篡改攻击防护，即对请求包进行签名保护，防止被恶意篡改 3、针对核心系统，集权系统和三级及以上系统，需要进行防重放和防篡改攻击防护
- 风险影响：攻击者截取请求和响应，利用重放、篡改方式持续进行漏洞探测，可能产生代码执行、权限绕过、敏感信息泄露等各类风险
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》


## AUTH 身份鉴别（10 项）

### AUTH-001 验证码功能安全测试 ｜风险等级：高危
- 功能要求：1、应用系统登录认证（包含应用后台管理）应包含图形码验证功能。考虑用户体验，可弹性使用图形码（例如首次验证不使用图形码，在验证失败后，再辅助图形码进行验证）或滑块验证等 2、验证码应在服务后端进行验证 3、完善验证码检验机制，避免空参数绕过
- 风险影响：存在账密爆破风险，攻击者破解窃取应用账号，导致应用敏感信息泄露
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》

### AUTH-002 账号注册、密码找回及改密功能安全测试 ｜风险等级：高危
- 功能要求：1、应通过管理员在后台进行账号开通（集成IAM系统，可通过IAM改密模块操作），如在账号自行注册的场景里，要有复杂验证码或其他人机验证功能，防止账号批量注册行为 2、在改密时需要输入原密码进行验证 3、密码找回需要多因素验证
- 风险影响：存在账号批量注册攻击、账号猜测攻击风险，攻击者破解窃取应用账号，导致应用敏感信息泄露
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》

### AUTH-003 双因素验证功能安全测试 ｜风险等级：中危
- 功能要求：1、业务系统应接入IAM，并开启双因素验证（面向C端由安全主任审核通过可例外） 2、每个账号只允许每隔60s发送一次短信、邮件验证信息 3、每个账号发送短信、邮件验证信息在120s有效 4、每个账号发送短信、邮件验证码要求不小于6位
- 风险影响：短信及邮件模块安全机制不完善可能存在短信轰炸、资源枯竭、验证码预测与复用的攻击风险，攻击者通过不断重放短信及邮件发生请求，进行短信及邮件炸弹攻击
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》 《GBT22239-2019信息安全技术网络安全等级保护基本要求》

### AUTH-004 首次登录强制改密安全测试 ｜风险等级：中危
- 功能要求：1、用户使用初始化密码（包括管理员分配的密码）在进行首次登录时，应用应强制要求用户修改密码，密码长度一般不应小于8个字符且应包含大小写字母、数字和特殊字符中的至少三种。 2、对于三级系统，如果初始账号长时间未登录（建议在1个月内），应采取账号冻结措施。
- 风险影响：存在数据泄露风险。应用可能出现大量账号使用同一初始密码，攻击者可以通过社工等手段获取初始密码后导致数据泄露。
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》

### AUTH-005 身份验证失败处理安全测试 ｜风险等级：中危
- 功能要求：系统登录失败提示语句因进行统一模糊说明
- 风险影响：存在应用敏感信息泄露风险，系统登录失败提示语句不一致，攻击者通过提示语句进行账密爆破，破解窃取应用账号从而导致应用敏感信息泄露
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### AUTH-006 账号多次失败登录锁定安全测试 ｜风险等级：高危
- 功能要求：1、应用应设置账号多次登录失败锁定账号，并且锁定账号在一定时间内自动解锁 2、三级系统应在5次输错图形码情况下，也应联动锁定账号，并且锁定账号在一定时间内自动解锁
- 风险影响：存在账密爆破风险，攻击者破解窃取应用账号，导致应用敏感信息泄露
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》

### AUTH-007 登录密码禁止明文展示安全测试 ｜风险等级：中危
- 功能要求：在登录界面、改密界面、账号注册等场景下，输入密码时应禁止使用明文显示，可允许短时间明文查看密码，或明文查看部分密码位数
- 风险影响：存在账密泄露风险，攻击者可以通过恶意软件远程录屏截图等手段获取账号密码，导致应用敏感信息泄露
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### AUTH-008 强密码策略安全测试 ｜风险等级：高危
- 功能要求：1、系统应具备足够强度的密码策略，要求密码长度不小于8位，且应包含大小写字母、数字和特殊字符中的至少三种 2、三级系统应具备伪强密码检测模块
- 风险影响：存在敏感信息泄露风险，用户使用如123456等弱口令密码，这类密码极易被攻击者猜测破解从而获取应用账号权限，导致应用敏感信息泄露
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》

### AUTH-009 长期不登录账号锁定安全测试 ｜风险等级：中危
- 功能要求：应用账号半年不登录自动锁定
- 风险影响：存在应用敏感信息泄露风险，应用可能出现不再使用却仍保有高度权限的幽灵账号，由于没有人定期更新该类账号的密码，账号可能会被攻击者破解窃取，导致应用敏感信息泄露
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》

### AUTH-010 用户标识、角色标识安全测试 ｜风险等级：高危
- 功能要求：不应在传输过程中带用户标识（如userid、供应商id）、角色标识，应通过cookie、token进行交互验证，userid、供应商id在后端和对应cookie、token进行关联授权 若有传输需要，项目组须提供必要性说明。
- 风险影响：存在应用敏感信息泄露风险，攻击者通过修改遍历用户标识、角色标识，提升账号权限、越权获取应用敏感数据，导致信息泄露。
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》


## PER 权限管理（3 项）

### PER-001 权限控制功能安全测试 ｜风险等级：严重
- 功能要求：1、系统应具备账号赋权功能，可控制任意账户可以访问与操作的资源 2、系统至少具备管理员账户、普通账户两类角色，该两类角色权限应分开 3、管理员应具备对账号进行冻结（锁定）功能
- 风险影响：存在权限滥用与恶意操作风险，管理员权限过大时，容易引发权限滥用的问题；存在敏感信息泄露风险，管理员用户没有账号冻结功能，可能导致发生账号泄露事件时不能及时止损。
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》

### PER-002 角色越权安全测试 ｜风险等级：高危
- 功能要求：1. 系统需实现基于角色的访问控制（RBAC）机制，严格划分不同角色的操作权限边界 2. 验证不同角色间的权限隔离有效性，普通用户无法执行管理员级操作
- 风险影响：未管控将导致低权限用户/未授权人员获取高权限操作权限，篡改系统配置、窃取敏感数据、删除核心业务数据
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》

### PER-003 数据越权安全测试 ｜风险等级：高危
- 功能要求：1. 系统需实现数据级权限控制，确保用户仅能访问/操作自身权限范围内的数据 2. 验证同级别角色数据隔离有效性：相同角色的不同用户，无法访问、查看、修改、删除其他用户的个人数据
- 风险影响：未管控将导致用户数据泄露、数据被非法篡改/删除，敏感信息外泄，侵犯用户隐私
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》


## SESS 会话安全（6 项）

### SESS-001 Cookie安全测试 ｜风险等级：中危
- 功能要求：Cookie应启用Secure标识以及HttpOnly标识 如果一个Cookie缺少Secure标识，则浏览器允许它通过非加密的HTTP连接发送，增加了数据泄露的风险。
- 风险影响：1、存在数据篡改的风险，如果Cookie缺少Secure标识，攻击者可以截获Cookie，冒充用户身份 2、存在数据泄露的风险，如果Cookie缺少HttpOnly标识，JavaScript可读取Cookie值
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### SESS-002 CORS跨域资源共享安全测试 ｜风险等级：高危
- 功能要求：系统应正确配置CORS，避免出现CORS（跨域资源共享）源验证失败问题
- 风险影响：错误的CORS配置会导致数据泄露、未授权的资源访问等风险，恶意跨域请求可能被用于执行应用层面的DDOS攻击
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### SESS-003 会话登出功能安全测试 ｜风险等级：中危
- 功能要求：系统功能应经过登录认证后才能访问，退出登录后会话应立即销毁
- 风险影响：会话未正确销毁可被重用，存在会话劫持风险
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### SESS-004 会话超时安全测试 ｜风险等级：中危
- 功能要求：所有应用程序应实现会话空闲超时机制。建议设置为30min超时。
- 风险影响：设备长期登录账号可能出现账号泄露风险
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》

### SESS-005 会话标识安全测试 ｜风险等级：中危
- 功能要求：1、会话标识应具备唯一性和随机性。 2、会话标识应从一个可信系统，如在应用服务端创建，而不是在客户端创建。
- 风险影响：存在敏感信息泄露风险，攻击者猜测破解会话标识从而获取应用账号权限，导致应用敏感信息泄露
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### SESS-006 会话固定安全测试 ｜风险等级：高危
- 功能要求：在用户提供的认证信息（如用户名和密码）、相应的权限级别发生变化时，服务器端应重新生成新会话标识，并强制失效之前的旧会话标识。
- 风险影响：存在敏感信息泄露风险，攻击者利用会话标识固定不变机制，劫持他人会话
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

## BUS 业务安全（5 项）

### BUS-001 文件上传安全测试 ｜风险等级：高危
- 功能要求：文件上传功能应包含以下安全策略： 系统应不存在可以直接部署网页脚本的文件上传功能。 上传文件前应经过有效的身份验证。 文件上传的校验应服务端进行。 文件上传功能应存在有效的后缀白名单限制。
- 风险影响：存在数据泄露、内网被进一步扩大范围攻击的风险，攻击者可利用文件上传漏洞远程部署木马
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### BUS-002 文件下载安全测试 ｜风险等级：高危
- 功能要求：在进行文件下载时，应确保下载权限设计合理（如：未授权下载、可读权限能绕过下载文件、下载非预期目录里文件）
- 风险影响：任意文件下载导致源代码和敏感配置泄露
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### BUS-003 多线程竞争条件安全测试 ｜风险等级：中危
- 功能要求：应用应不存在多线程竞争条件缺陷。对于业务端条件竞争的防范，一般的方法是设置锁。
- 风险影响：攻击者利用该漏洞可在积分不足情况执行恶意下单兑换商品。
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》

### BUS-004 重要数据类型验证安全测试 ｜风险等级：中危
- 功能要求：应用业务逻辑功能应不存在数据类型验证缺陷。如将相关业务逻辑功能中的各个关键参数赋值为空值、0、负数值、超大数字、字符串等出现的各类非预期结果。
- 风险影响：攻击者绕过篡改机制，利用应用数据类型验证缺陷，如将积分数值、商品数量、userid进行删除或篡改，导致获取非预期恶意收益或权限。
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》

### BUS-005 业务流程非预期执行安全测试 ｜风险等级：中危
- 功能要求：所在业务流程执行过程中，应具备防护机制来保证其遵循正常流程设计执行，所在的业务流程阶段不能被用户恶意非预期执行。
- 风险影响：攻击者非预期恶意打乱执行业务流程，可导致恶意操作。
- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》


## AUD 日志审计（3 项）

### AUD-001 基础日志审计功能安全测试 ｜风险等级：中危
- 功能要求：1、应用应具备安全审计日志记录功能 2、安全审计日志应无法被修改、删除 3、安全审计日志应提供日志数据统计、查询功能
- 风险影响：存在安全事件无法定位溯源的风险，缺少安全审计日志，会导致出现安全事件后，不能及时溯源定位问题账号以及问题功能

- 功能要求：1、安全审计日志应包含：登录日志、操作日志 （1）登录日志：记录用户登录、登出、及登录结果等，应包含登录时间、登录IP、登录账号和登录结果 （2）操作日志:记录用户对功能模块访问及操作信息

- 测试依据：《GBT22239-2019信息安全技术网络安全等级保护基本要求》 《中华人民共和国网络安全法（2025 修正，自 2026-01-01 施行）》


## DATA 数据安全（3 项）

### DATA-001 加密传输安全测试 ｜风险等级：高危
- 功能要求：应用应对数据通信过程中的敏感数据进行加密。应使用SHA-256及以上哈希算法，TLS 1.2及以上版本。
- 风险影响：存在敏感信息泄露风险，数据在传输过程中，如果未经加密，就容易被攻击者截获。
- 测试依据：《数据安全法》 《个人信息保护法》 《GBT22239-2019信息安全技术网络安全等级保护基本要求》

### DATA-002 数据脱敏展示安全测试 ｜风险等级：严重
- 功能要求：应用页面敏感信息应默认脱敏显示，如身份证件号码、手机号码等。可以通过点击"显示详情"按钮查看完整信息。
- 风险影响：存在敏感信息泄露风险，攻击者可以通过恶意软件远程录屏截图等手段获取敏感信息
- 测试依据：《数据安全法》 《个人信息保护法》 《GBT22239-2019信息安全技术网络安全等级保护基本要求》

### DATA-003 数据导出安全测试 ｜风险等级：高危
- 功能要求：1、二级系统导出信息包含个人信息，应支持文件加密导出 2、三级系统及三级以上系统数据导出应通过审批流程管控，同时日志记录导出行为
- 风险影响：可能存在敏感数据泄露风险，攻击者获取应用权限后通过数据导出，获取大量个人信息、重要业务数据
- 测试依据：《数据安全法》 《个人信息保护法》


## VULN 常见漏洞（20 项）

### VULN-001 HTTPS安全测试 ｜风险等级：高危
- 功能要求：1、要求使用https协议进行加密传输。 2、要求使用1.2或更高版本的TLS协议。低版本的TLS协议由于设计缺陷，存在已知的安全漏洞。 3、添加安全策略头: HSTS、CSP、X-Content-Type-Options等。
- 风险影响：存在信息泄露的风险：未使用HTTPS协议或使用低版本TLS协议，传输数据可被中间人截获
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### VULN-002 HTTP高危请求方法安全测试 ｜风险等级：中危
- 功能要求：要求应用服务使用安全的http请求方法，PUT、DELETE、TRACE等应该被禁止使用。
- 风险影响：存在信息泄露的风险，不安全的http请求方法可能造成恶意文件上传、XST攻击等安全问题
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### VULN-003 弱口令安全测试 ｜风险等级：中危
- 功能要求：系统应具备足够强度的密码策略，要求密码长度不小于8位，且应包含大小写字母、数字和特殊字符中的至少三种
- 风险影响：弱口令漏洞使得攻击者能够相对容易地获取账户的访问权限，进而可能对系统、应用程序或敏感数据造成未经授权的访问
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》 《OWASP Top 10:2025》

### VULN-004 XSS安全测试 ｜风险等级：高危
- 功能要求：XSS(Cross Site Scripting)跨站脚本漏洞，应用应防范反射型、存储型和DOM型XSS攻击。
- 风险影响：存在对造成用户负面影响的风险，攻击者利用XSS漏洞窃取敏感信息，如用户的cookie、会话令牌等
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》 《OWASP Top 10:2025》

### VULN-005 XXE安全测试 ｜风险等级：高危
- 功能要求：XXE（XML External Entity Injection）漏洞，应用应禁用外部实体解析。
- 风险影响：存在数据泄露风险，攻击者利用XXE漏洞执行SSRF攻击，获取内部系统的敏感信息
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### VULN-006 CSRF安全测试 ｜风险等级：高危
- 功能要求：CSRF（Cross-Site Request Forgery）跨站请求伪造，应用应实施CSRF Token或其他防护机制。
- 风险影响：存在应用信息泄露风险，攻击者利用用户的登录状态，在用户不知情的情况下执行攻击者预设的操作
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》 《OWASP Top 10:2025》

### VULN-007 SSRF安全测试 ｜风险等级：高危
- 功能要求：应用软件应不存在SSRF漏洞。SSRF(Server-Side Request Forgery)服务器端请求伪造漏洞。
- 风险影响：SSRF漏洞可导致敏感信息泄露、内网探测、远程命令执行等
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》 《OWASP Top 10:2025》

### VULN-008 SQL注入安全测试 ｜风险等级：严重
- 功能要求：SQL注入漏洞，应用应使用参数化查询或ORM框架防范SQL注入。
- 风险影响：成功的SQL注入攻击可以从数据库中读取敏感数据、修改数据库数据，对数据库执行管理操作
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》 《OWASP Top 10:2025》

### VULN-009 LDAP注入安全测试 ｜风险等级：中危
- 功能要求：LDAP(Lightweight Directory Access Protocol)注入，应对用户输入进行转义过滤。
- 风险影响：成功的LDAP注入攻击可以读取、修改LDAP目录树中的敏感数据
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》 《OWASP Top 10:2025》

### VULN-010 文件包含安全测试 ｜风险等级：高危
- 功能要求：应用应不存在文件包含漏洞(LFI/RFI)。
- 风险影响：存在信息泄露的风险，攻击者通过恶意构造的输入，诱使服务器加载并执行非预期的文件
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### VULN-011 反序列化安全测试 ｜风险等级：高危
- 功能要求：应用应不存在反序列化漏洞，禁止反序列化不可信来源的数据。
- 风险影响：存在信息系统被远控的风险，攻击者通过构造恶意序列化数据实现远程命令执行
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### VULN-012 远程命令执行安全测试 ｜风险等级：严重
- 功能要求：应用应不存在远程命令执行漏洞(RCE)。
- 风险影响：攻击者通过向目标系统发送恶意命令，远程执行系统命令或代码，可能导致系统被完全控制
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### VULN-013 暗链安全测试 ｜风险等级：低危
- 功能要求：应用不应包含隐蔽链接(hidden links)，暗链可能链接到恶意网站。
- 风险影响：存在信息泄露的风险，被植入恶意链接可能造成用户信息泄露
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### VULN-014 开发框架、中间件安全测试 ｜风险等级：中危
- 功能要求：开发框架与中间件应安全配置，禁用调试端点和管理接口的公网访问。
- 风险影响：存在敏感信息泄露风险，攻击者可利用框架漏洞获取服务器权限
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### VULN-015 未授权访问安全测试 ｜风险等级：严重
- 功能要求：应用软件应不存在常见的未授权访问漏洞，如Spring Boot Actuator未授权访问漏洞，Swagger未授权访问漏洞等。
- 风险影响：存在敏感信息泄露风险
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### VULN-016 敏感信息泄露安全测试 ｜风险等级：严重
- 功能要求：应用软件应不存在下列敏感信息泄露风险 1、源代码泄露 2、配置文件泄露 3、错误信息泄露 4、API文档泄露 5、日志文件泄露
- 风险影响：存在应用服务敏感信息泄露风险
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### VULN-017 处理用时安全测试 ｜风险等级：低危

- 风险影响：存在应用软件无法正常提供服务的风险，影响用户体验
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

### VULN-018 目录遍历安全测试 ｜风险等级：高危
- 功能要求：应用应严格限制访问权限，仅允许访问预设的合法目录及文件，禁止跨目录访问系统敏感资源。
- 风险影响：攻击者可遍历读取系统敏感文件，包括用户账号密码、数据库连接信息、配置文件等
- 测试依据：《OWASP Web 安全测试指南 WSTG v4.2》

# 附录 B：59 项逐项操作手册（测试手法 + 判定标准）

> 每项给出「怎么测」（curl/浏览器/工具步骤）与「判定标准」（符合 / 不符合，缺条件据实标未测试）。风险等级为项固有值，不随测试结果改变。


## PORT 端口安全

### PORT-001 应用端口服务安全测试 ｜风险等级：严重
- 测试手法：对目标 IP 做端口扫描：`nmap -sT -Pn -p- <host>` 或 `nmap --top-ports 1000`；对开放端口逐个识别服务(`nmap -sV`)；重点排查 Tomcat AJP(8009)、Redis(6379)、MySQL(3306)、Actuator、Druid、SSH、管理端口是否对外可直连。
- 判定标准：符合：对外仅暴露业务必需端口(如 80/443)，管理端/中间件端口不可直接访问；AJP 已隔离。不符合：存在高危端口(数据库/缓存/管理端/AJP)可直连。


## BASIC 基线安全

### BASIC-001 应用软件及相关组件版本基线安全测试 ｜风险等级：高危
- 测试手法：识别技术栈与组件版本：看响应头 Server/X-Powered-By、前端 bundle 特征、后端框架指纹(RuoYi/Spring)、nginx/tomcat 版本；对照 CVE 库核实是否为已知漏洞版本。
- 判定标准：符合：框架/中间件/组件均为受支持且无已知高危 CVE 的版本。不符合：使用 EOL 或存在已知高危漏洞的组件版本。


## FRON 前端安全

### FRON-001 前端JS库软件版本安全测试 ｜风险等级：中危
- 测试手法：抓取前端 JS 依赖(package 特征/bundle 内库版本)，用 `retire.js` 或对照 Snyk/CVE 核查 jQuery、lodash、axios、Vue 等库版本。
- 判定标准：符合：JS 库均为无已知漏洞版本。不符合：存在含已知漏洞的低版本 JS 库。

### FRON-002 前端泄露风险测试（如JS） ｜风险等级：严重
- 测试手法：下载所有前端 JS/source map(`index.*.js`、`*.js.map`)，grep：私钥/公钥(BEGIN PRIVATE KEY、MII…)、AK/SK、密码、内网 IP、接口地址、注释中的敏感信息；检查是否泄露 sourcemap。
- 判定标准：符合：前端不含私钥、硬编码凭据、内网地址、敏感注释，未泄露 sourcemap。不符合：前端泄露私钥/密钥对/凭据/内网信息(如 RuoYi 默认 512 位 RSA 公私钥硬编码即严重不符合)。


## TRAN 传输安全

### TRAN-001 软件版本信息泄漏安全测试 ｜风险等级：中危
- 测试手法：检查响应头是否泄露版本：`curl -I`，看 Server、X-Powered-By；错误页/默认页是否暴露中间件版本号。
- 判定标准：符合：隐藏 Server 版本号、无版本信息泄露。不符合：nginx/tomcat 等版本号在响应头或错误页明文暴露。

### TRAN-002 防重放攻击、防篡改功能安全测试 ｜风险等级：高危
- 测试手法：抓登录/关键写操作请求，重放同一请求包看是否被服务端接受(防重放)；篡改参数/签名看是否被校验(防篡改)；检查是否有 timestamp/nonce/签名机制。
- 判定标准：符合：关键接口具备时间戳+nonce+签名，重放/篡改被拒。不符合：无防重放/防篡改，请求可被原样重放或篡改后仍被接受。


## AUTH 身份鉴别

### AUTH-001 验证码功能安全测试 ｜风险等级：高危
- 测试手法：查看登录是否需要验证码；抓登录接口看 `captchaEnabled`；尝试空验证码/不带验证码提交、同一验证码多次使用、验证码是否可绕过。
- 判定标准：符合：登录启用有效验证码(图形/滑块等)且一次一用、服务端校验。不符合：验证码关闭(captchaEnabled:false)或可绕过/复用，登录接口可无验证码提交。

### AUTH-002 账号注册、密码找回及改密功能安全测试 ｜风险等级：高危
- 测试手法：测试注册、密码找回、改密流程：是否开放自助注册(内网系统应关闭)；找回流程是否存在账号枚举、验证码轰炸、越权重置；改密是否校验原密码。
- 判定标准：符合：无开放注册(或注册受控)，找回/改密校验充分、无越权、改密需原密码。不符合：开放注册、找回流程可枚举/越权重置、改密不校验原密码。

### AUTH-003 双因素验证功能安全测试 ｜风险等级：中危
- 测试手法：检查登录是否支持并强制第二因素(短信/OTP/UKey)；管理员账号是否强制 MFA。
- 判定标准：符合：重要/管理员账号启用强制双因素。不符合：无 MFA 或未对高权限账号强制启用。

### AUTH-004 首次登录强制改密安全测试 ｜风险等级：中危
- 测试手法：用初始/默认口令账号首次登录，看是否强制修改初始密码。
- 判定标准：符合：首次登录强制改密。不符合：可继续使用初始/默认口令，不强制修改。

### AUTH-005 身份验证失败处理安全测试 ｜风险等级：中危
- 测试手法：输入错误用户名 vs 正确用户名+错误密码，比较错误提示与响应时间是否可区分(账号枚举)；观察是否返回过多细节。
- 判定标准：符合：失败提示统一泛化(如"用户名或密码错误")，不可枚举。不符合：提示可区分账号存在与否，或泄露过多信息。

### AUTH-006 账号多次失败登录锁定安全测试 ｜风险等级：高危
- 测试手法：对同一账号连续多次错误登录，观察是否触发锁定/延时/验证码升级。
- 判定标准：符合：达到阈值(如 5 次/10 分钟)后锁定或强制验证码。不符合：无失败锁定，可无限尝试(结合验证码关闭=可爆破)。

### AUTH-007 登录密码禁止明文展示安全测试 ｜风险等级：中危
- 测试手法：查看密码输入框 type 是否 password(掩码显示)；抓包看口令是否明文传输(应加密)。
- 判定标准：符合：密码框掩码显示、口令加密传输。不符合：明文显示或明文传输。

### AUTH-008 强密码策略安全测试 ｜风险等级：高危
- 测试手法：测试设置弱口令(如 123456/纯数字/短口令)是否被系统拒绝；检查是否强制长度+复杂度。
- 判定标准：符合：强制强口令策略(长度≥8、多字符类型)。不符合：允许设置弱口令。

### AUTH-009 长期不登录账号锁定安全测试 ｜风险等级：中危
- 测试手法：检查长期未登录账号是否自动锁定/停用(需策略或长期观察，通常查配置)。
- 判定标准：符合：存在长期未登录自动锁定机制。不符合/未测试：无该机制或无法在测试窗口内验证(据实标未测试)。

### AUTH-010 用户标识、角色标识安全测试 ｜风险等级：高危
- 测试手法：检查用户标识/角色标识是否可预测或可枚举(自增 ID)、是否在前端/接口过度暴露；`getInfo`/`getRouters` 返回内容。
- 判定标准：符合：标识不可枚举、最小暴露。不符合：用户/角色标识可枚举或接口过度暴露权限信息。


## PER 权限管理

### PER-001 权限控制功能安全测试 ｜风险等级：严重
- 测试手法：验证 RBAC：是否有角色-菜单-权限绑定、能否启停用户、越权功能是否受控；用 admin 查看权限模型。
- 判定标准：符合：具备完整 RBAC、权限受控、可启停账号。不符合：无有效权限模型或权限失控。

### PER-002 角色越权安全测试 ｜风险等级：高危
- 测试手法：用低权限账号访问高权限功能 URL/接口(垂直越权)，看是否被拒；篡改角色参数。
- 判定标准：符合：低权限账号访问高权限功能被拒(403)。不符合：可垂直越权。缺普通账号时标未测试。

### PER-003 数据越权安全测试 ｜风险等级：高危
- 测试手法：用 A 用户 Token 访问 B 用户数据(改 id/越权查询,水平越权)，看是否做数据归属校验。
- 判定标准：符合：跨用户/跨部门数据访问被拒。不符合：可水平越权读写他人数据。缺普通账号时标未测试。


## SESS 会话安全

### SESS-001 Cookie安全测试 ｜风险等级：中危
- 测试手法：检查会话 Cookie 属性：`HttpOnly`、`Secure`、`SameSite`；Token 是否存 localStorage(易被 XSS 窃取)。
- 判定标准：符合：会话 Cookie 设 HttpOnly+Secure+SameSite。不符合：缺 HttpOnly/Secure，或 Token 存 localStorage 且全站 HTTP。

### SESS-002 CORS跨域资源共享安全测试 ｜风险等级：高危
- 测试手法：`curl -X OPTIONS -H 'Origin: https://evil.example' -H 'Access-Control-Request-Method: POST' <api>`，看 `Access-Control-Allow-Origin` 是否反射任意源、是否同时 `Allow-Credentials:true`。
- 判定标准：符合：CORS 白名单受限，不反射任意源，或不与凭据同开。不符合：ACAO 反射任意 Origin 或 `*` 且 `Allow-Credentials:true`。

### SESS-003 会话登出功能安全测试 ｜风险等级：中危
- 测试手法：登录后登出，再用旧 Token 请求接口，看服务端是否已失效该会话。
- 判定标准：符合：登出后旧 Token 立即失效(服务端销毁)。不符合：登出后旧 Token 仍可用。

### SESS-004 会话超时安全测试 ｜风险等级：中危
- 测试手法：登录后空闲一段时间再操作，观察会话是否超时失效；检查超时时长是否合理。
- 判定标准：符合：存在合理空闲超时(如 30 分钟)且到期失效。不符合：无超时或超时过长。

### SESS-005 会话标识安全测试 ｜风险等级：中危
- 测试手法：检查会话标识(JWT/SessionID)是否足够随机、不可预测；是否含敏感明文。
- 判定标准：符合：会话标识随机不可预测(如随机 UUID)。不符合：可预测或含敏感明文。

### SESS-006 会话固定安全测试 ｜风险等级：高危
- 测试手法：登录前后对比会话标识是否变化(防会话固定)：登录成功后应重新签发新 Token。
- 判定标准：符合：每次登录重新签发会话标识。不符合：登录前后会话标识不变(会话固定风险)。


## KEY 密钥安全

### KEY-001 密钥存储安全测试 ｜风险等级：严重
- 测试手法：检查密钥存储位置：前端 JS、配置文件、代码仓库是否硬编码密钥/私钥；密钥长度与算法强度。
- 判定标准：符合：密钥存服务端安全存储、长度足够(RSA≥2048)。不符合：前端/代码硬编码密钥或私钥，或密钥过短(如 512 位 RSA)。

### KEY-002 AK/SK专项安全测试 ｜风险等级：严重
- 测试手法：grep 前端/接口/仓库中的 AccessKey/SecretKey(云 AK/SK、OSS、短信、支付密钥)是否泄露。
- 判定标准：符合：无 AK/SK 泄露。不符合：前端或响应中泄露云/第三方 AK/SK。


## ERR 异常处理

### ERR-001 应用组件错误信息安全测试 ｜风险等级：中危
- 测试手法：对接口发异常输入(错误类型、超长、非法编码)、访问不存在路径，看是否返回堆栈/框架错误/SQL 报错等内部信息。
- 判定标准：符合：错误统一友好提示，不泄露堆栈/框架/SQL 细节。不符合：回显堆栈/框架级错误(如"Padding error in decryption"、500 异常明细)。


## BUS 业务安全

### BUS-001 文件上传安全测试 ｜风险等级：高危
- 测试手法：测试文件上传：上传非白名单类型(.jsp/.php/.html)、改 Content-Type/双扩展名/00 截断，看服务端是否校验并拒绝；上传目录是否可执行。
- 判定标准：符合：服务端白名单校验、拒危险类型、上传目录不可执行。不符合：可上传可执行/脚本文件或校验仅在前端。缺上传入口时标未测试。

### BUS-002 文件下载安全测试 ｜风险等级：高危
- 测试手法：测试文件下载/导出接口：改 filename/path 参数做路径穿越(../)、越权下载他人文件。
- 判定标准：符合：下载做路径/权限校验，无穿越无越权。不符合：可路径穿越或越权下载。

### BUS-003 多线程竞争条件安全测试 ｜风险等级：中危
- 测试手法：对涉及金额/库存/抽奖/额度的接口并发请求(如 `ab`/多线程脚本)，看是否存在竞争条件导致超发/重复。
- 判定标准：符合：关键操作有锁/幂等控制。不符合：并发下出现超发/重复扣减。缺相应业务入口时标未测试。

### BUS-004 重要数据类型验证安全测试 ｜风险等级：中危
- 测试手法：对关键参数做类型/边界/负数/超长/篡改测试(金额改负、数量越界)，看服务端校验。
- 判定标准：符合：服务端严格校验数据类型与边界。不符合：可提交非法/越界数据被接受。

### BUS-005 业务流程非预期执行安全测试 ｜风险等级：中危
- 测试手法：跳过业务流程步骤/乱序调用接口(如未支付直接完成、跳过审批)，看是否强制流程顺序。
- 判定标准：符合：服务端强制业务流程顺序与状态机。不符合：可越序执行绕过关键步骤。


## AUD 日志审计

### AUD-001 基础日志审计功能安全测试 ｜风险等级：中危
- 测试手法：检查是否记录操作日志/登录日志；查看日志管理菜单与记录数。
- 判定标准：符合：具备操作日志+登录日志且持续记录。不符合/未测试：无审计日志或无权限查看(据实标注)。

### AUD-002 基础日志审计内容安全测试 ｜风险等级：中危
- 测试手法：检查日志字段是否完整：账号、客户端 IP、时间、模块、操作类型、请求方法、结果状态。
- 判定标准：符合：日志含账号/IP/时间/模块/操作/方法/状态等关键字段。不符合：字段缺失(如无操作人或无 IP)。

### AUD-003 基础日志审计存储安全测试 ｜风险等级：中危
- 测试手法：检查日志留存时长与存储安全(是否可被普通用户删除/篡改、留存是否≥6 个月)。
- 判定标准：符合：日志留存≥6 个月且防篡改。不符合/未测试：留存不足或无法验证(据实标注)。


## DATA 数据安全

### DATA-001 加密传输安全测试 ｜风险等级：高危
- 测试手法：抓包看敏感数据(口令、身份证、发票、Token)是否加密传输；是否全站 HTTPS；有无明文 HTTP。
- 判定标准：符合：全站 HTTPS，敏感数据加密传输。不符合：明文 HTTP 传输敏感数据(如 443 端口跑明文 HTTP、无 TLS)。

### DATA-002 数据脱敏展示安全测试 ｜风险等级：严重
- 测试手法：在列表/详情/导出中检查敏感字段是否脱敏：手机号、身份证、银行账号、税号、地址、姓名；横向滚动露出全部列核查。
- 判定标准：符合：敏感字段默认脱敏(掩码)。不符合：敏感字段明文全量展示(如发票查询 200+ 条税号/银行账号/VIN 明文)。

### DATA-003 数据导出安全测试 ｜风险等级：高危
- 测试手法：测试导出功能：能否批量导出未脱敏敏感数据；导出是否有审批/审计/数量限制。
- 判定标准：符合：导出脱敏或受审批+审计+限量。不符合：可批量导出未脱敏敏感数据无管控。


## VULN 常见漏洞

### VULN-001 HTTPS安全测试 ｜风险等级：高危
- 测试手法：检查是否强制 HTTPS、HTTP 是否 301 跳 HTTPS、是否配置 HSTS；TLS 版本与弱套件。
- 判定标准：符合：强制 HTTPS+HSTS，无弱 TLS。不符合：无 HTTPS 或未强制、无 HSTS、存在弱协议套件。

### VULN-002 HTTP高危请求方法安全测试 ｜风险等级：中危
- 测试手法：`curl -X OPTIONS -i` 看 Allow；测试 PUT/DELETE/TRACE/PATCH 是否被服务端接受。
- 判定标准：符合：仅允许必要方法(GET/POST)，禁用 PUT/DELETE/TRACE。不符合：开放危险 HTTP 方法。

### VULN-003 弱口令安全测试 ｜风险等级：中危
- 测试手法：对登录/关键账号尝试常见弱口令(admin/123456 等)字典；检查管理员口令强度。
- 判定标准：符合：无弱口令，账号口令强。不符合：存在可爆破的弱口令账号。

### VULN-004 XSS安全测试 ｜风险等级：高危
- 测试手法：在输入点(搜索/表单/富文本/URL 参数)注入 `<script>`/`<img onerror>` 等，看是否被转义或执行(反射/存储/DOM XSS)。
- 判定标准：符合：输出编码/框架自动转义，无 XSS。不符合：payload 被执行(反射/存储/DOM XSS)。

### VULN-005 XXE安全测试 ｜风险等级：高危
- 测试手法：对接收 XML 的接口提交含外部实体(DOCTYPE+ENTITY)的 payload，看是否解析外部实体(读文件/SSRF)。
- 判定标准：符合：禁用外部实体解析或不接收 XML(纯 JSON)。不符合：解析外部实体导致文件读取/SSRF。

### VULN-006 CSRF安全测试 ｜风险等级：高危
- 测试手法：检查写操作是否依赖可被跨站携带的 Cookie 且无 CSRF Token/SameSite；结合 CORS 判断。
- 判定标准：符合：用 Authorization 头 Token(非自动携带)或有 CSRF Token/SameSite。不符合：仅靠 Cookie 且无防护(叠加宽松 CORS 更严重)。

### VULN-007 SSRF安全测试 ｜风险等级：高危
- 测试手法：对含 URL/地址参数的接口(图片拉取、webhook、导入)传入内网地址/元数据地址(169.254.169.254)，看是否发起请求。
- 判定标准：符合：对外发请求做白名单/内网禁访。不符合：可让服务端访问内网/云元数据(SSRF)。

### VULN-008 SQL注入安全测试 ｜风险等级：严重
- 测试手法：在查询/参数注入 `'`、`or 1=1`、时间盲注 payload，看报错/延时/结果异常；用 sqlmap 验证。
- 判定标准：符合：参数化查询，无注入。不符合：存在 SQL 注入(报错/布尔/时间盲注可利用)。

### VULN-009 LDAP注入安全测试 ｜风险等级：中危
- 测试手法：对涉及目录服务查询的接口注入 LDAP 特殊字符(`*)(&`)，看是否影响查询逻辑。
- 判定标准：符合：过滤 LDAP 特殊字符或不涉及 LDAP。不符合：存在 LDAP 注入。

### VULN-010 文件包含安全测试 ｜风险等级：高危
- 测试手法：对含文件路径/模板参数的接口测试本地/远程文件包含(../、php://、http://)。
- 判定标准：符合：无动态包含或做白名单。不符合：可包含本地/远程文件。

### VULN-011 反序列化安全测试 ｜风险等级：高危
- 测试手法：检查是否有反序列化入口(Java 原生序列化、fastjson、Jackson 多态);对可疑接口投递恶意序列化 payload。
- 判定标准：符合：无不安全反序列化(禁用危险特性)。不符合：存在可利用反序列化(如 fastjson 高危版本)。

### VULN-012 远程命令执行安全测试 ｜风险等级：严重
- 测试手法：对可能拼接系统命令的接口注入 `;id`、`|whoami`、反引号，看是否命令执行。
- 判定标准：符合：无命令拼接或严格过滤。不符合：存在远程命令执行。

### VULN-013 暗链安全测试 ｜风险等级：低危
- 测试手法：检查页面源码是否被植入隐藏暗链/黑帽 SEO 链接(display:none 的外链)。
- 判定标准：符合：无暗链。不符合：页面存在隐藏恶意外链。

### VULN-014 开发框架、中间件安全测试 ｜风险等级：中危
- 测试手法：识别开发框架/中间件默认页、示例接口、管理后台(如 RuoYi 演示数据、swagger、druid)是否暴露且有默认配置。
- 判定标准：符合：无框架默认页/示例暴露、无默认配置。不符合：暴露框架默认组件或默认配置(如 nginx 版本暴露、示例接口开放)。

### VULN-015 未授权访问安全测试 ｜风险等级：严重
- 测试手法：直接访问需鉴权的接口/管理端点(actuator、druid、swagger、导出、后台)不带 Token，看是否可未授权访问。
- 判定标准：符合：受保护资源均需鉴权(未授权返回 401/403)。不符合：存在未授权可访问的敏感接口/管理端点。

### VULN-016 敏感信息泄露安全测试 ｜风险等级：严重
- 测试手法：综合检查敏感信息泄露：接口返回、错误信息、监控端点(server/cache)、备份文件(.bak/.git/.svn/.zip)、目录列表。
- 判定标准：符合：无敏感信息泄露。不符合：泄露服务器信息/内网 IP/源码/备份/监控数据(如 monitor/server 泄露 OS/JVM/内网 IP)。

### VULN-017 处理用时安全测试 ｜风险等级：低危
- 测试手法：测量敏感操作(登录/查询)响应时间差异，判断是否可用时间侧信道推断(如账号存在性)。
- 判定标准：符合：无可利用的时间差。不符合：响应时间可用于枚举/推断。

### VULN-018 目录遍历安全测试 ｜风险等级：高危
- 测试手法：对文件/路径参数与静态目录测试目录遍历(`../../etc/passwd`、`..%2f`)，看能否越目录读文件或列目录。
- 判定标准：符合：无目录遍历、关闭目录列表。不符合：可目录遍历读取任意文件或列目录。

### VULN-019 CRLF注入安全测试 ｜风险等级：中危
- 测试手法：在会跟随进响应头的参数注入 `%0d%0a` 换行，看能否注入响应头/拆分响应(CRLF)。
- 判定标准：符合：过滤 CR/LF，无头注入。不符合：存在 CRLF 注入/HTTP 响应拆分。

### VULN-020 URL重定向安全测试 ｜风险等级：中危
- 测试手法：对含 redirect_uri/url/target 等跳转参数传入外部域名，看是否无校验跳转(开放重定向)。
- 判定标准：符合：跳转做白名单校验或不涉及。不符合：可跳转任意外部域名(开放重定向,可用于钓鱼)。
