# 应用安全检测报告（标准模板）技能

把「客户标准模板」固化为可复用的检测方法论 + 报告生成流水线。适用于任意 Web/应用系统的
经授权应用安全检测。报告统一为 **13 类 / 55 项**，格式与客户 `assets/标准模板_参考.docx` 一致。

## 何时使用
- 用户要求对某个应用/网址做「应用检测 / 应用安全检测 / 上线前安全检测 / 安全评估」；
- 用户要求按此标准模板出正式检测报告（DOCX/PDF）。

## 铁律（真实性 + 低影响 + 脱敏）——任何情况下不得违反
1. **证据优先**：每一项结论都必须有真实证据（HTTP 请求/响应、页面内容、前端 JS、API 响应、
   浏览器操作或截图）。没有证据不许写结论。
2. **未做即"未测试"**：没有实际验证的项一律标 `未测试`，并在「截图说明」写明原因（缺对照账号 /
   生产环境非破坏性约束 / 经隧道无法验证 等）。绝不把推测写成已确认漏洞。
3. **设计能力 ≠ 已利用**：如 GLUE(Shell/Python) 可执行代码、版本号对应已知 CVE 等，只能写
   `设计性风险，非本次实际利用结果`，不得声称已实现 RCE / 已利用 CVE。
4. **客户端可见 ≠ 可伪造/可利用**：如 Cookie 内含疑似散列，只写实测现象，不得断言"MD5/无盐/
   可伪造"，除非有充分证据。
5. **低影响**：不做破坏性 SQL 注入 / SSRF / RCE / XXE / 反序列化验证；不高强度爆破；不创建/
   执行任何任务（GLUE/Shell/Python/Node）；不删除或修改生产数据。
6. **脱敏交付**：报告及提交物中**绝不出现**口令、完整 Cookie 值、Bearer/令牌、GitHub PAT、
   AK/SK、cpolar 配置、32 位散列原文或任何非必要的原始敏感证据。内网地址可脱敏后作为观察写入。
7. **标准现行**：参考标准每次执行前重新核对是否有更新版本，写明名称+版本/修订日期，不用模糊表述。
   仅列与被测对象行业相关的标准。
8. **数量一致**：报告必须严格 55 项、13 类，编号与 `assets/items_55.json` 完全一致。

## 标准检测项框架（13 类 / 55 项）
`assets/items_55.json` 是环境无关的标准清单，字段：`测试编号 / 风险名称 / 功能要求 /
风险影响 / 测试依据 / 风险等级参考`。分类分布（不可改动）：

```
PORT 端口安全 1   BASIC 基线安全 1   FRON 前端安全 2   TRAN 传输安全 2
AUTH 身份鉴别 10  PER 权限管理 2    SESS 会话安全 6   Key 密钥安全 2
INFO 异常处理 1   BUS 业务安全 5    AUD 日志审计 3    DATA 数据安全 3
VULN 常见漏洞 17                                        合计 = 55
```

> 说明：模板原表格「风险等级」列写的是**定级规则文本**（按等保级别定高/中），因此本技能把
> 实际风险等级放在 `findings`（判定时给出），标准清单只保留定级参考。

## 执行流程

### 1. 前置输入（向用户确认，缺则据实标注）
- 目标网址 / 是否已授权；
- 访问方式：目标常为内网域名，云端不可直达 → 需用户在其挂 VPN 的机器上建 **cpolar HTTP 隧道**
  并固定 Host 头（见下方「内网穿透」）；
- 账号：管理员账号（深测用）；如有**普通用户账号**用于越权/数据隔离(PER)验证，缺则相关项标未测试；
- 交付去向（本例：推送 GitHub `change1238/32` main，直接推送不建 PR）。

### 2. 内网穿透（cpolar 经验，重要）
- **HTTP 隧道会终止 TLS**：目标真实证书/TLS 版本/密码套件经隧道无法独立验证 → TLS 相关项
  (如 TRAN/DATA/VULN 中 TLS 项) 必须标注此限制，据实标未测试。
- cpolar `tcp` 命令行不支持转发到远程主机（会打到本机 127.0.0.1）→ 用 **HTTP 隧道 + host-header**：
  `cpolar http https://<域名>:443 -host-header=<域名> -config="<cpolar.yml 路径>"`
  （host-header 不能带 `:443`，否则后端 ELB 按 server_name 匹配失败回 404）。
- cpolar token 常存于 `-config` 指定的 `cpolar.yml`（`findstr /s /i /m authtoken *.yml` 定位）。

### 3. 检测（黑盒 → 认证态深测）
- 黑盒：端口/指纹/版本、未认证可见信息、响应头、HTTP 方法、登录页 JS。
- 认证态：登录后逐类走查 13 类 55 项；抓取真实请求/响应、页面、API、截图取证到 `evidence/`。
- 逐项把「实际证据」对照「测试依据/判定标准」判定 `符合/不符合/未测试/不涉及`，**边测边落证据**。

### 4. 生成报告
把逐项结论写入 `findings.json`（可从 `assets/findings.template.json` 起改），填 `meta.json`
（从 `assets/meta.example.json` 起改），截图放 `evidence/`，然后：

```bash
# 生成前必须先自检（编号覆盖 / 字段完整 / 结果与等级合法 / 敏感信息扫描）
python3 assets/validate.py --items assets/items_55.json \
        --findings findings.json --meta meta.json
# 校验通过后再生成
python3 assets/gen_report.py --items assets/items_55.json \
        --findings findings.json --meta meta.json \
        --evidence ./evidence --out 报告.docx
# 转 PDF（需 libreoffice）
libreoffice --headless --convert-to pdf 报告.docx
```

### 5. 交付前终检（务必自动执行，别等用户发现）
- `validate.py` 校验通过（55 项、字段无空缺、不符合项有等级+修复建议、无敏感信息告警）；
- 解析生成的 DOCX：详表数 = 55、编号去重 = 55、分类分布与上表一致；
- 从逐项动态统计结果分布与不符合等级分布，写进「风险信息」（生成器已自动统计，勿手填）；
- 参考标准逐条核对为现行版本；
- 仅提交指定文件（如 `xxx_应用安全检测报告.docx` / `.pdf`），不 `git add .`，不提交脚本/证据/
  cookies/配置/凭据；如需新增改动用新 commit，不 amend。

## 报告格式规范（与模板一致）
- **封面**：系统名称 / 应用安全检测报告 / 初测 / 委托单位 / 测试人员 / 测试日期 / 机构名称。
- **一、检测概述**：背景介绍 / 参考标准 / 适用说明 / 工作原则 / 检测内容 / 检测工具 / 检测声明 /
  变更记录。
- **二、检测结果**：检测信息 / 风险信息 / 风险汇总（**仅列不符合项**，字段：序号 / 风险章节(3.x.y) /
  风险名称 / 风险等级 / 复测结果<留空>）。
- **三、检测详情**：13 类，每检测项一张两列表，字段顺序固定：
  `测试编号 / 风险名称 / 风险等级 / 功能要求 / 风险影响 / 测试依据 / 测试结果 / 截图说明 /
  修复建议 / 备注`。三级标题**只写风险名称**（如「应用端口服务安全测试」），**不带编号前缀**
  （编号在表格「测试编号」行里，勿写成「PORT-001 应用端口服务安全测试」）。
- **风险等级只有两级：`高` / `中`**（本模板无「严重」「低」）。判定不符合的项等级只能是 高 或 中；
  如沿用其它四级口径，须先折算：严重→高、低→中。汇总/详表均以单字展示。

## 参考标准（现行版本，编制前须复核）
见 `assets/meta.example.json` 的 `参考标准` 字段。截至本技能编写时的现行版本：
OWASP WSTG **v4.2**、OWASP Top 10:**2025**、PTES、ISO/IEC 27001:**2022**、
GB/T 22239-**2019**（2025 复审继续有效）、GB/T 30279-2020、GB/T 30276-2020、
GB/T 40856-2021、《汽车数据安全管理若干规定(试行)》、UN R155 / R156、
《网络安全法》(2025 修正，2026-01-01 施行)、《数据安全法》、《个人信息保护法》。
> 车联网/汽车类标准仅在被测对象属该行业时保留，否则删除；换行业须替换为对应行业标准。

## 资产清单
- `assets/标准空白模板.docx` — 由生成器产出的空白标准模板（格式基准，全项占位为"未测试"；
  未内嵌任何历史工程截图/数据，可安全分发）。
- `assets/items_55.json` — 13 类 / 55 项标准检测项清单（环境无关）。
- `assets/findings.template.json` — 逐项结论填写模板（默认全"未测试"占位）。
- `assets/meta.example.json` — 报告元信息示例（项目/标准/工具/变更记录）。
- `assets/gen_report.py` — 报告生成器（items + findings + meta → DOCX）。
- `assets/validate.py` — 生成前后自检（覆盖/字段/等级/敏感信息扫描）。
